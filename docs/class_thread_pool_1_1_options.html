Detailed Description

This class allows users to specify the number of threads to be used in parallel algorithms.

These constants can be used to specify the number of threads in parallel algorithms.

Enumerator

Disable multi-threading, executing tasks sequentially.

Integer ThreadPool::Options::actual_nthreads ( const Integer user_num_threads)
inlinestatic

This function interprets the user-specified number of threads and returns the actual number of threads to be used. If the preprocessor flag THREADPOOL_SINGLE_THREADED is defined, it always returns 0, indicating that multi-threading is disabled.

Integer ThreadPool::Options::get_nthreads ( ) const

Note
This function may return 0, which means that multi-threading shall be switched off entirely. If an algorithm receives this value, it should revert to a sequential implementation. In contrast, if get_nthreads() == 1, the parallel algorithm version shall be executed with a single thread.
Returns

◆ nthreads()

Options & ThreadPool::Options::nthreads ( const Integer n = AUTO)
inline
Note
This setting is ignored if the preprocessor flag THREADPOOL_SINGLE_THREADED is defined. Then, the number of threads is set to 0 and all tasks revert to sequential algorithm implementations. The same can be achieved at runtime by passing n = 0. In contrast, passing n = 1 causes the parallel algorithm versions to be executed with a single thread. Both possibilities are mainly useful for debugging.
Parameters
[in]nThe desired number of threads.
